► SCRIPT DE CORRECTIONS POUR LE PROJET « LIBFT » DE 42

► Ceci n’est pas un script de correction automatique qui lancera des séries de tests et vous dira si la fonction est OK ou non.

Il s’agit d’un support pour le(la) correcteur(trice) qui aurait décidé de corriger chaque fonction par lui(elle)-même, une par une.

Ce scrit ajoute et supprime un « main » à la fonction correspondante, si elle existe, afin de compiler un exécutable.
C’est ensuite au(à la) correcteur(trice) d’entrer le(s) paramètre(s) qu’il(elle) souhaite et d’étudier les résultats obtenus, éventuellement en les comparant aux résultats des fonctions originelles, pour la partie obligatoire.

► UTILISATION

=> Modifiez la variable « CHEM » au tout début du script pour indiquer le chemin relatif vers le dossier contenant vos fonctions, puis entrez la commande « sh sh_include_mains », suivi de :

- « all » pour compiler tous les exécutables de toutes les fonctions.
- « nom de la fonction sans le préfixe ‘ft_’ » (ex : « atoi ») pour compiler l’exécutable correspondant à la fonction souhaitée.
- « bonus » pour compiler les exécutables correspondant aux fonctions de la partie bonus.
- « erase » pour supprimer tous les exécutables et fichiers tests créés.

=> Une fois les exécutables créés, lancez-les sans paramètre pour afficher des informations concernant la fonction à tester ainsi que les paramètres à passer à l’exécutable.
